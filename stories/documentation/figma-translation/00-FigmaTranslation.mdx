# Figma Translation System

## What This Is

This documentation suite is the **translation layer** between Figma's design vocabulary and the Discourser Design System's implementation vocabulary (Panda CSS, Park UI, Ark UI).

It serves two audiences simultaneously:

- **Kai (Design Engineer Agent)** тАФ reads these docs via Storybook MCP to make accurate token and component decisions when converting `get_design_context` output into Design Implementation Specs
- **Human developers** тАФ onboarding from Tailwind/Shadcn, or joining a project that consumes DDS

## Two Token Systems, Both Valid

Before reading anything else, understand this: DDS exposes **two coexisting token vocabularies**. Both are real Panda CSS tokens defined in `panda.config.ts`. Do not mix them within a single component.

| System | Origin | When to use | Example |
|---|---|---|---|
| **M3 Semantic** | `src/preset/semantic-tokens.ts` | All new component code | `bg="surface.container"`, `color="onSurface"` |
| **Park UI Aliases** | `panda.config.ts` semantic tokens | Component recipe compatibility | `bg="canvas"`, `color="fg.default"`, `borderColor="border"` |

The page root (`globalCss` in `panda.config.ts`) uses Park UI aliases. Component recipes mix both. **For Kai generating implementation code: default to M3 Semantic. Use Park UI aliases only when you see them in an existing recipe you are extending.**

## Three-Layer Token Architecture

DDS separates infrastructure from aesthetic so the design language can be swapped without touching component code.

### Layer 1: Infrastructure (Never Changes)

Park UI and Panda CSS define **naming conventions and scale structure**. These names are stable regardless of which design language is active:

- Radii: `l1` (2px) ┬╖ `l2` (6px) ┬╖ `l3` (8px) ┬╖ `full`
- Shadows: `xs` ┬╖ `sm` ┬╖ `md` ┬╖ `lg` ┬╖ `xl` ┬╖ `2xl`
- Spacing scale: `xxs` ┬╖ `xs` ┬╖ `sm` ┬╖ `md` ┬╖ `lg` ┬╖ `xl` ┬╖ `xxl` ┬╖ `xxxl`
- Park UI aliases: `fg.default` ┬╖ `fg.muted` ┬╖ `fg.subtle` ┬╖ `canvas` ┬╖ `border`

### Layer 2: Design Language (Swappable)

Material Design 3 provides the **values** mapped to Layer 1 names via `src/languages/material3.language.ts`. Change `src/languages/index.ts` to swap the entire aesthetic.

- M3 tonal palettes тЖТ Radix color scales (1тАУ12)
- M3 type scale тЖТ `textStyle` tokens (`bodyMedium`, `titleLarge`, etc.)
- M3 elevation levels тЖТ shadow levels (level0тАУ5) тЖТ semantic aliases (xsтАУxl)
- M3 surface container system тЖТ `surface.container.*` tokens

### Layer 3: Components (Derived)

Component recipes reference **only Layer 1 tokens via the `colorPalette` system**. They never hardcode hex values or design-language-specific names:

```tsx
// тЬЕ Component code тАФ decoupled from design language
<Button colorPalette="primary" variant="solid">Submit</Button>

// Recipe uses:  bg: 'colorPalette.solid.bg'   color: 'colorPalette.solid.fg'
// Resolves to:  bg: primary   color: onPrimary   (at runtime from colorPalette prop)
```

## Quick Reference Rules for Kai

```tsx
// тЬЕ Semantic tokens use dot-notation
bg="surface.container"       // NOT bg="surfaceContainer"
color="onSurface.variant"    // NOT color="onSurfaceVariant"
borderColor="outline.variant"

// тЪая╕П These three are flat camelCase (exceptions, not dot-notation)
bg="surfaceVariant"          // NOT bg="surface.variant"
bg="inverseSurface"          // NOT bg="inverse.surface"
color="inverseOnSurface"     // NOT color="inverse.onSurface"

// тЬЕ Radii always l1/l2/l3/full тАФ never sm/md/lg from preset
rounded="l2"   // buttons, inputs, badges
rounded="l3"   // cards, dialogs, popovers
rounded="full" // avatars, switches, pills

// тЬЕ Shadows always semantic aliases тАФ never level0-5
shadow="sm"    // level2 тАФ cards
shadow="md"    // level3 тАФ dialogs
shadow="lg"    // level4 тАФ drawers

// тЬЕ colorPalette on components тАФ never manual bg+color
<Button colorPalette="primary" variant="solid">   // тЬЕ
<Button bg="primary" color="onPrimary">           // тЭМ

// тЬЕ DDS variant names
variant="solid"     // filled (not "filled")
variant="outline"   // outlined (not "outlined")
variant="subtle"    // tinted (not "tonal")
variant="plain"     // ghost (not "ghost" or "text")
```
