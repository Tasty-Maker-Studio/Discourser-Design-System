# Token Resolution Reference

How Figma variables, legacy library names, and raw hex values map to the actual DDS token system.

---

## Token System Architecture

The DDS uses a three-layer token stack. Each layer has a specific job:

```
Layer 1 — Primitives
  Raw tonal steps: primary/0 … primary/100, secondary/0 … secondary/100, etc.
  Source: src/languages/material3.language.ts
  Use in code: Never directly. These are the raw palette.

Layer 2 — Semantic
  M3 color roles: primary, onSurface, surface/container, outline, etc.
  Source: Figma "Semantic" collection → Panda CSS semantic tokens
  Use in code: Always prefer these. They're theme-aware and adapt to dark mode.

Layer 3 — DDS (Panda CSS)
  Panda token paths consumed in component recipes and sx props.
  Examples: bg="surface", color="onSurface", borderColor="outline.variant"
  Source: styled-system/tokens (generated from panda.config.ts)
```

**Rule:** Use Layer 2 (Semantic) tokens in code. Only drop to Primitives when you
need a specific tonal step for a custom one-off (e.g., an icon that must always be
`primary/80` regardless of semantic role).

## Quick Decision Guide

**What token should I use for...?**

### Text

| Purpose | Token | Panda CSS |
|---|---|---|
| Body text on any surface | `onSurface` | `color="onSurface"` |
| Secondary / helper text | `onSurface/variant` | `color="onSurface.variant"` |
| Text on a primary-colored button | `onPrimary` | `color="onPrimary"` |
| Text on a secondary button | `onSecondary` | `color="onSecondary"` |
| Text on page background | `onBackground` | `color="onBackground"` |
| Error message text | `onError/container` | `color="onError.container"` |
| Label on chips/tags | `onSurface/variant` | `color="onSurface.variant"` |

### Backgrounds

| Purpose | Token | Panda CSS |
|---|---|---|
| Page background | `background` | `bg="background"` |
| Card / dialog | `surface/container` | `bg="surface.container"` |
| Subtle input background | `surface/container/low` | `bg="surface.container.low"` |
| Elevated card | `surface/container/high` | `bg="surface.container.high"` |
| Primary action button | `primary` | `bg="primary"` or `colorPalette="primary"` |
| Secondary action button | `secondary` | `bg="secondary"` or `colorPalette="secondary"` |
| Tooltip / snackbar | `inverse/surface` | `bg="inverseSurface"` |

### Borders

| Purpose | Token | Panda CSS |
|---|---|---|
| Input / card border | `outline/variant` | `borderColor="outline.variant"` |
| Divider / separator | `outline/variant` | `borderColor="outline.variant"` |
| Strong focus ring / panel border | `outline` | `borderColor="outline"` |

### Actions

| Purpose | Token | Panda CSS |
|---|---|---|
| Primary CTA button | `colorPalette="primary"` on `<Button>` | (component handles bg + text) |
| Destructive action | `colorPalette="error"` on `<Button>` | (component handles bg + text) |
| Secondary/neutral action | `colorPalette="secondary"` on `<Button>` | — |
| Icon button fill | `onSurface` or `onSurface/variant` | `color="onSurface"` |

### Legacy Tonal Scale (Primary/500 etc.)

Older design files sometimes used a **100–900 tonal scale** (Chakra / Tailwind convention)
instead of the M3 **0–100 scale**. The approximate conversion:

| Legacy name | M3 equivalent | Hex (light) |
|---|---|---|
| `Primary/500` | `primary/50` → use `primary` Semantic | `#4C662B` |
| `Primary/400` | `primary/40` | `#3F6900` |
| `Accent/500` | `secondary/50` → use `secondary` Semantic | `#586249` |
| `Stone/60%/500` | `neutral/50` at 60% opacity | `#777771` + alpha |
| `Stone/60%/400` | `neutral/40` at 60% opacity | `#5E5F59` + alpha |

> **Note:** These legacy names appear in the Figma audit for node `66:2739`
> (Conversation Prelaunch). The frame was built against an older library version.
> When rebinding these nodes, target the **Semantic** collection — not Primitives.

## Radii Reference

DDS uses **Park UI semantic radii** (`l1`, `l2`, `l3`, `full`) as the canonical tokens for all component recipes.

> The M3 base names (`extraSmall`, `small`, `medium`, `large`, `extraLarge`) come from the Figma
> `Spacing & Shape` variable collection and exist in `styled-system/tokens`, but are **not used in
> component recipes**. When you encounter these names in a Figma audit, map them to the DDS
> canonical token on the right.

| Figma token | Value | M3 base name (pipeline only) | DDS recipe token |
|---|---|---|---|
| `radii/none` | `0px` | `none` | `rounded="none"` |
| `radii/extraSmall` | `4px` | `extraSmall` | `rounded="l1"` (2px — closest) |
| `radii/small` | `8px` | `small` | `rounded="l3"` (8px) or `rounded="l2"` (6px) |
| `radii/medium` | `12px` | `medium` | `rounded="l3"` (8px — closest) |
| `radii/large` | `16px` | `large` | `rounded="l3"` (8px — closest) |
| `radii/extraLarge` | `28px` | `extraLarge` | No direct match — use `rounded="full"` for pills |
| `radii/full` | `9999px` | `full` | `rounded="full"` |

**Canonical usage summary:**
- `rounded="l1"` (2px) — checkboxes, breadcrumbs, subtle rounding
- `rounded="l2"` (6px) — buttons, inputs, badges, tooltips, textarea, tabs
- `rounded="l3"` (8px) — cards, dialogs, popovers, toasts, nav menus
- `rounded="full"` (9999px) — pills, avatars, switches, spinners, toggle tracks

## Elevation (Shadow) Reference

DDS shadows use a **two-layer system**.

**Layer 1 — Base tokens** (`level0`–`level5`): M3 elevation values registered via `transform.ts`.
Do **not** use these directly in component code.

**Layer 2 — Semantic tokens** (`xs`, `sm`, `md`, `lg`, `xl`, `2xl`): Defined in `src/preset/shadows.ts`.
These are the tokens to use in component recipes and custom styling.

| M3 Level | Semantic token | Shadow value | Use case |
|---|---|---|---|
| `level0` | `shadow="none"` | `none` | Flat surfaces, inline content |
| `level1` | `shadow="xs"` | `0px 1px 2px rgba(0,0,0,0.3), 0px 1px 3px 1px rgba(0,0,0,0.15)` | Cards at rest, contained buttons |
| `level2` | `shadow="sm"` | `0px 1px 2px rgba(0,0,0,0.3), 0px 2px 6px 2px rgba(0,0,0,0.15)` | Cards on hover, raised buttons |
| `level3` | `shadow="md"` | `0px 4px 8px 3px rgba(0,0,0,0.15), 0px 1px 3px rgba(0,0,0,0.3)` | **Dialogs, dropdowns, popovers** |
| `level4` | `shadow="lg"` | `0px 6px 10px 4px rgba(0,0,0,0.15), 0px 2px 3px rgba(0,0,0,0.3)` | Navigation drawers, modal sheets |
| `level5` | `shadow="xl"` | `0px 8px 12px 6px rgba(0,0,0,0.15), 0px 4px 4px rgba(0,0,0,0.3)` | FABs, tooltips, snackbars |

```tsx
// ✅ Correct: semantic shadow token
<Box shadow="md">Dialog content</Box>

// ❌ Wrong: base token directly (don't use level0–level5 in component code)
<Box shadow="level3">Dialog content</Box>

// ❌ Wrong: raw CSS string
<Box boxShadow="0px 4px 8px 3px rgba(0,0,0,0.15)...">...</Box>
```
