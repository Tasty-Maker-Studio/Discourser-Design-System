# Shadows & Border Radius Translation Guide

This page translates shadow and border-radius values from Tailwind CSS and Figma to Discourser's elevation and radii system.

---

## Shadow Mapping (Elevation)

Discourser exposes shadows through a **three-layer architecture**:

1. **Base tokens** (`level0`–`level5`) — Raw M3 values, defined in `transform.ts`. **Never use directly in components.**
2. **Semantic tokens** (`xs`, `sm`, `md`, `lg`, `xl`, `2xl`) — Defined in `src/preset/shadows.ts`. **Use these in components.**
3. **Component usage** — Components use the semantic tokens.

| Source (Tailwind) | Discourser | M3 Base Level | Visual Effect | Usage |
|---|---|---|---|---|
| `shadow-none` | `shadow="none"` | level0 | Flat, no shadow | Inline elements, flush surfaces |
| `shadow-sm` | `shadow="xs"` | level1 | Subtle lift | Subtle cards, chips |
| `shadow` | `shadow="sm"` | level2 | Light elevation | Default cards, panels |
| `shadow-md` | `shadow="md"` | level3 | Moderate lift | Raised elements, dialogs, popovers |
| `shadow-lg` | `shadow="lg"` | level4 | Strong elevation | Navigation drawers, modal sheets |
| `shadow-xl` | `shadow="xl"` | level5 | Maximum elevation | FABs, tooltips, snackbars |
| `shadow-2xl` | `shadow="2xl"` | level5 | Maximum elevation | Large overlays |

**Important:** Do not use the base level tokens (`level0`–`level5`) directly in component code. Always use the semantic aliases (`xs`, `sm`, `md`, `lg`, `xl`, `2xl`).

### M3 Elevation Strategy

Material Design 3 uses **surface container tint + shadow** for elevation. Shadow alone misses the tint layer:

- **Level 0:** `bg="surface"` or `bg="surface.container.lowest"` (no shadow)
- **Level 1:** `bg="surface.container"` + `shadow="xs"`
- **Level 2:** `bg="surface.container"` + `shadow="sm"`
- **Level 3:** `bg="surface.container.high"` + `shadow="md"`
- **Level 4:** `bg="surface.container.highest"` + `shadow="lg"`
- **Level 5:** `bg="surface.container.highest"` + `shadow="xl"`

```tsx
// ✅ CORRECT: M3-style elevation with surface container + shadow
<Card.Root bg="surface.container" shadow="sm">
  Card content
</Card.Root>

// ❌ LESS IDEAL: Shadow alone without surface tint
<Card.Root shadow="sm">
  Card content (missing M3 surface tint)
</Card.Root>
```

### Inset Shadow

| Source | Discourser | Notes |
|---|---|---|
| `shadow-inner` | `shadow="inset"` | Inset shadow for recessed effects (inputs, wells) |

## Border Radius Mapping

Discourser uses **Park UI semantic radii** (`l1`, `l2`, `l3`, `full`) as the canonical tokens for all components. These are defined in `panda.config.ts`.

### Canonical Radii (Use These in Components)

| Token | Value | Usage |
|---|---|---|
| `l1` | 2px | Subtle rounding — checkboxes, breadcrumbs |
| `l2` | 6px | Standard rounding — inputs, badges, tooltips, textarea, tabs, buttons |
| `l3` | 8px | Large rounding — cards, dialogs, popovers, toasts, nav menus |
| `full` | 9999px | Pills, avatars, switches, spinners — fully circular |

**Always use `l1`/`l2`/`l3`/`full` for component code.** The M3 base tokens (`extraSmall`, `small`, `medium`, `large`, `extraLarge`) come from the token pipeline but are **not used in recipes**.

### Tailwind → Discourser Radii

| Source (Tailwind) | Discourser | Notes |
|---|---|---|
| `rounded-none` | `rounded="none"` | No rounding |
| `rounded-sm` | `rounded="l1"` | 2px — subtle rounding |
| `rounded` / `rounded-md` | `rounded="l2"` | 6px — standard rounding (most components) |
| `rounded-lg` | `rounded="l3"` | 8px — cards, dialogs |
| `rounded-full` | `rounded="full"` | Circular — avatars, pills, icon buttons |

**Note:** The `@pandacss/preset-panda` also provides `xs`, `sm`, `md`, `lg`, `xl`, `2xl` radii tokens. These can be used for one-off custom styling, but **do not use them in component recipes** — use `l1`/`l2`/`l3` for consistency.

### Rounded Corners (Directional)

| Property | Example | Effect |
|---|---|---|
| `borderTopLeftRadius` | `borderTopLeftRadius="l2"` | Top-left corner |
| `borderTopRightRadius` | `borderTopRightRadius="l2"` | Top-right corner |
| `borderBottomLeftRadius` | `borderBottomLeftRadius="l2"` | Bottom-left corner |
| `borderBottomRightRadius` | `borderBottomRightRadius="l2"` | Bottom-right corner |
| `borderTopRadius` | `borderTopRadius="l2"` | Top-left + Top-right |
| `borderBottomRadius` | `borderBottomRadius="l2"` | Bottom-left + Bottom-right |
| `borderLeftRadius` | `borderLeftRadius="l2"` | Top-left + Bottom-left |
| `borderRightRadius` | `borderRightRadius="l2"` | Top-right + Bottom-right |

## Common Patterns

### Card with Default Elevation

```tsx
// Figma: Card with subtle elevation (M3 level 1-2)
<Card.Root bg="surface.container" shadow="sm" rounded="l3">
  <Card.Header>Title</Card.Header>
  <Card.Body>Content</Card.Body>
</Card.Root>
```

### Dialog with Strong Elevation

```tsx
// Figma: Modal dialog (M3 level 3)
<Dialog.Content bg="surface.container.high" shadow="md" rounded="l3">
  Dialog content
</Dialog.Content>
```

### Button with Default Rounding

```tsx
// Figma: Button with 6px border-radius
<Button rounded="l2">
  Click me
</Button>
```

### Avatar (Circular)

```tsx
// Figma: Circular avatar
<Avatar.Root rounded="full">
  <Avatar.Image src="..." />
</Avatar.Root>
```

## Common Mistakes

### ❌ Using Pixel Values

```tsx
// ❌ WRONG: Hardcoded pixel value
<Box style={{ boxShadow: '0 4px 6px rgba(0,0,0,0.1)', borderRadius: '8px' }}>
  Content
</Box>
```

```tsx
// ✅ CORRECT: Token values
<Box shadow="md" rounded="l3">
  Content
</Box>
```

---

### ❌ Using Base-Level Shadow Tokens in Components

```tsx
// ❌ WRONG: Using level tokens directly (level0–level5 are internal base tokens)
<Card.Root shadow={'level2'}>
  Card content
</Card.Root>
```

**Why wrong:** `level0`–`level5` are base tokens. Components should use the semantic aliases.

```tsx
// ✅ CORRECT: Semantic shadow token
<Card.Root shadow="sm">
  Card content
</Card.Root>
```

---

### ❌ Shadow Without Surface Container

```tsx
// ❌ WRONG: Shadow alone for elevation
<Card.Root shadow="md">
  Card content
</Card.Root>
```

**Why wrong:** M3 uses surface tint + shadow for elevation.

```tsx
// ✅ CORRECT: Surface container + shadow
<Card.Root bg="surface.container.high" shadow="md">
  Card content
</Card.Root>
```

---

### ❌ Using Preset Radii Scale for Components

```tsx
// ❌ WRONG: Using preset numeric scale in component code
<Card.Root rounded="md">
  <Button rounded="sm">Action</Button>
</Card.Root>
```

**Why wrong:** `md`, `sm` from `@pandacss/preset-panda` are not the canonical DDS radii. Use `l1`/`l2`/`l3`.

```tsx
// ✅ CORRECT: Semantic l-scale for components
<Card.Root rounded="l3">
  <Button rounded="l2">Action</Button>
</Card.Root>
```

---

### ❌ Excessive Rounding for UI Elements

```tsx
// ❌ WRONG: Full rounding on wide rectangular button
<Button rounded="full" px="8">
  Wide button
</Button>
```

**Why wrong:** `rounded="full"` is for circular elements (avatars, icon buttons). Wide buttons look distorted.

```tsx
// ✅ CORRECT: Appropriate rounding for button
<Button rounded="l2" px="8">
  Wide button
</Button>

// ✅ CORRECT: Full rounding for icon button (square)
<IconButton rounded="full">
  <Icon />
</IconButton>
```

- `@pandacss/preset-panda` — Additional radii tokens available but not preferred for component recipes
