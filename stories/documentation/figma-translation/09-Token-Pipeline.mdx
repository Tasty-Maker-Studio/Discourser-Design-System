# Token Pipeline: Soup to Nuts

This document maps the complete lifecycle of a design token — from its origin in Material Theme Builder through to the JSON mapping structure Kai (the Discourser.ai BMAD Design Engineer Agent) uses to generate Design Implementation Specs.

**Audiences:**
- **Kai** — understand where tokens come from, what files are authoritative, and how to resolve a Figma variable name to a Panda CSS prop
- **DDS maintainers** — understand every transformation step so you can update, extend, or debug the pipeline
- **Discourser.ai developers** — understand how `figma-codex.json` + `token-name-mapping.json` work together as Kai's resolution layer

## Step 1: Material Theme Builder

**What it does:** Takes a seed color and generates a complete M3 color system — six tonal palettes (primary, secondary, tertiary, neutral, neutralVariant, error) each with 13 tonal steps (0–100), plus two sets of semantic color roles (light theme and dark theme).

**DDS seed color:** `#63A002` (TastyMakers green)

**Output:** `docs/material-theme.json` — the raw export. This file is committed to the repo as a reference but is **not** the source of truth for code.

**Why it matters:** M3 tonal steps are the primitives everything else derives from. A semantic token like `primary` (light: `#4C662B`) maps to `primary/40` in the tonal palette. This relationship is encoded in the DTCG transform scripts.

## Step 2: DDS Source of Truth — `material3.language.ts`

**File:** `src/languages/material3.language.ts`

**What it is:** The single TypeScript implementation of `DesignLanguageContract` that drives everything. All token values in DDS ultimately trace back to this file.

**Key sections:**

```typescript
material3Language = {
  colors: {
    primary:   { 0: '#000000', 10: '#102000', ... 100: '#FFFFFF' },
    secondary: { ... },   // 6 palettes × 13 tonal steps
    tertiary:  { ... },
    neutral:   { ... },
    neutralVariant: { ... },
    error:     { ... },
  },

  semantic: {                          // LIGHT THEME
    primary: '#4C662B',
    onPrimary: '#FFFFFF',
    primaryContainer: '#CDEDA3',
    surface: '#F9FAEF',
    surfaceContainer: '#EEEFE3',
    onSurface: '#1A1C16',
    outline: '#75796C',
    outlineVariant: '#C5C8BA',
    // ... 34 total semantic roles
  },

  semanticDark: { ... },               // DARK THEME (same 34 keys)

  typography: {
    fonts: {
      display: { family: '"Fraunces"...', figmaName: 'Fraunces', weightMap: {...} },
      body:    { family: '"Poppins"...',  figmaName: 'Poppins',  weightMap: {...} },
      mono:    { family: '"JetBrains Mono"...', weightMap: {...} },
    },
    scale: {
      displayLarge: {
        geometry: { fontSize: '57px', lineHeight: '64px', fontFamily: 'display' },
        defaultWeight: 'regular',
        weights: {
          regular:  { name: 'regular',  fontWeight: '400' },
          semiBold: { name: 'semiBold', fontWeight: '600' },
        },
      },
      // ... 15 scale steps total, each with geometry + weights
    },
  },

  spacing:   { none: '0px', xxs: '2px', xs: '4px', sm: '8px', md: '16px', ... },
  shape:     { radii: { none: '0px', extraSmall: '4px', small: '8px', ... } },
  elevation: { levels: { level0: 'none', level1: '0px 1px...', ... } },
  motion:    { durations: { fast: '100ms', normal: '200ms', ... }, easings: {...} },
  border:    { widths: { thin: '1px', medium: '2px', thick: '4px' } },
}
```

**To swap the entire design aesthetic:** Change one line in `src/languages/index.ts`:
```typescript
export { material3Language as activeLanguage } from './material3.language';
// ↑ Change this import to swap the entire design language
```

## Step 3: Figma Export — `pnpm figma:export`

**Script:** `scripts/export-figma-tokens.ts`

**Command:** `pnpm figma:export`

**What it produces:** Four files from zero hardcoded values — all data sourced from `material3.language.ts`.

### `tokens/export-to-figma/figma-variables.json`

Three Figma variable collections in a single file:

**Primitives collection** — tonal palette steps as flat color tokens:
```json
"primary/40": { "$type": "color", "$value": { "Value": "#3F6900" } },
"primary/50": { "$type": "color", "$value": { "Value": "#518500" } }
```

**Semantic collection** — M3 color roles with light/dark mode values:
```json
"primary":          { "$type": "color", "$value": { "Light": "#4C662B", "Dark": "#B1D18A" } },
"surface/container":{ "$type": "color", "$value": { "Light": "#EEEFE3", "Dark": "#1E201A" } },
"onSurface/variant":{ "$type": "color", "$value": { "Light": "#44483D", "Dark": "#C5C8BA" } }
```

> Note: Figma variable paths use slash notation (`surface/container`). Panda CSS uses dot notation (`surface.container`). `token-name-mapping.json` bridges this.

**Spacing & Shape collection** — numeric tokens:
```json
"spacing/md":      { "$type": "number", "$value": { "Value": 16 } },
"radii/small":     { "$type": "number", "$value": { "Value": 8 } },
"border/thin":     { "$type": "number", "$value": { "Value": 1 } },
"duration/normal": { "$type": "number", "$value": { "Value": 200 } }
```

### `tokens/export-to-figma/figma-text-styles.json`

51 text style entries — one per scale step × weight variant:
```json
{
  "name": "Headline/Small/SemiBold",
  "description": "dds:typography.scale.headlineSmall.weights.semiBold",
  "fontFamily": "Fraunces",
  "fontStyle": "SemiBold",
  "fontSize": 24,
  "lineHeightPx": 32,
  "letterSpacing": 0
}
```

> The `description` field is critical — it's the semantic bridge the Figma MCP reads when resolving typography to DDS tokens. Format: `dds:typography.scale.{scaleName}.weights.{weightName}`.

### `tokens/export-to-figma/figma-effect-styles.json`

6 elevation levels:
```json
"elevation": {
  "level2": {
    "value": "0px 1px 2px rgba(0,0,0,0.3), 0px 2px 6px 2px rgba(0,0,0,0.15)",
    "description": "Cards on hover, raised buttons (semantic: sm)"
  }
}
```

### `docs/token-name-mapping.json` ← also updated here

The export script also updates the token name mapping. See Step 6 for details.

## Step 4: Figma — The Design Tool

**File key:** `GaHmFfmvO4loUzuZS4TgEz`

**Plugin:** `figma-token-sync` (import workflow)

### DDS → Figma (push tokens)

```bash
pnpm figma:export
# Open figma-token-sync plugin in Figma
# Import Variables → tokens/export-to-figma/figma-variables.json
# Import Text Styles → tokens/export-to-figma/figma-text-styles.json
```

### Figma → DDS (pull changes back)

```bash
# In Figma: figma-token-sync plugin → Export Variables
# Downloads to tokens/imported-from-figma/
pnpm organize-tokens
# Review diffs, then:
pnpm build:panda
```

### Three Variable Collections in Figma

| Collection | Modes | Contains |
|---|---|---|
| **Primitives** | Value | Raw tonal steps — `primary/0` through `primary/100` for 6 palettes |
| **Semantic** | Light · Dark | 34 M3 color roles — `primary`, `surface/container`, `onSurface/variant`, etc. |
| **Spacing & Shape** | Value | Spacing, radii, border widths, motion durations |

> **Design rule:** Designers should bind nodes to **Semantic** collection variables, not Primitives. Semantic variables switch automatically in dark mode. Primitives are raw values — using them directly in designs breaks dark mode.

## Step 5: Figma Desktop MCP — What Kai Reads

**Tool:** `get_design_context` via Figma Desktop MCP (`http://127.0.0.1:3845/sse`)

When Kai calls `get_design_context` on a frame, the output contains:

### Color fills — `var()` syntax with fallback

```css
background-color: var(--colors-surface-container, #EEEFE3);
color: var(--colors-on-surface, #1A1C16);
border-color: var(--colors-outline-variant, #C5C8BA);
```

**How to resolve:** Strip `--colors-` prefix, convert remaining dashes to dots:
- `--colors-surface-container` → `surface.container`
- `--colors-on-surface-variant` → but wait — this is `onSurface.variant` (camelCase prefix preserved)
- Use `token-name-mapping.json` for the authoritative mapping

### Spacing — `var()` with named tokens

```css
padding: var(--spacing-md, 16px);
gap: var(--spacing-sm, 8px);
```

**Resolve:** Strip `--spacing-` → `md`, `sm` — these are Panda spacing tokens directly.

### Typography — text style description

```
style.description: "dds:typography.scale.headlineSmall.weights.semiBold"
style.fontFamily: "Fraunces"
style.fontWeight: 600
style.fontSize: 24
```

**How to resolve:** Strip `dds:` prefix → `typography.scale.headlineSmall.weights.semiBold`. This directly maps to:
- `textStyle="headlineSmall"` (geometry)
- `fontWeight="semibold"` (weight override, Panda token name)

### Component instances — CodeConnectSnippet

When a node is a DDS component instance, `get_design_context` returns the `figma-codex` data:
```
component: "Button"
variant props: { variant: "solid", size: "md", colorPalette: "primary" }
import: "import { Button } from '@discourser/design-system/Button'"
```

## Step 6: Token Resolution — The Two Lookup Files

Kai uses two JSON files as resolution layers when converting `get_design_context` output to code.

### `docs/token-name-mapping.json`

**Generated by:** `pnpm figma:export` (via `scripts/export-figma-tokens.ts`)

**Purpose:** Maps every Figma variable path → Panda CSS token path → CSS custom property → usage example. Also includes all 51 text style entries.

**Structure:**
```json
{
  "version": "0.25.3",
  "generatedFrom": ["tokens/export-to-figma/figma-variables.json", "..."],
  "semantic": [
    {
      "figmaPath": "Semantic/surface/container",
      "pandaToken": "surface.container",
      "cssProperty": "--colors-surface-container",
      "exampleUsage": "bg: 'surface.container'"
    }
  ],
  "spacingAndShape": [
    {
      "figmaPath": "Spacing & Shape/spacing/md",
      "pandaToken": "md",
      "pandaCategory": "spacing",
      "cssProperty": "--spacing-md",
      "exampleUsage": "p: 'md', gap: 'md'"
    },
    {
      "figmaPath": "Spacing & Shape/radii/small",
      "pandaToken": "small",
      "pandaCategory": "radii",
      "cssProperty": "--radii-small",
      "exampleUsage": "borderRadius: 'small'"
    }
  ],
  "effectStyles": [
    {
      "figmaEffectStyle": "elevation/level2",
      "pandaToken": "sm",
      "cssProperty": "--shadows-sm",
      "exampleUsage": "shadow: 'sm'"
    }
  ],
  "textStyles": [
    {
      "figmaTextStyle": "Headline/Small/SemiBold",
      "figmaDescription": "dds:typography.scale.headlineSmall.weights.semiBold",
      "ddsTokenPath": "typography.scale.headlineSmall.weights.semiBold",
      "pandaTextStyle": "headlineSmall",
      "fontFamily": "Fraunces",
      "fontStyle": "SemiBold",
      "fontWeight": 600,
      "fontSize": 24,
      "lineHeightPx": 32,
      "letterSpacing": 0,
      "isDefaultWeight": false,
      "exampleUsage": "textStyle: 'headlineSmall'"
    }
  ]
}
```

**Critical Mismatch Notes (documented in `docs/token-name-mapping.md`):**

| Figma path | Panda token | Note |
|---|---|---|
| `surface/variant` | `surfaceVariant` | Flat camelCase — NOT `surface.variant` |
| `onSurface/variant` | `onSurface.variant` | IS dot-notation (inconsistent with above) |
| `inverse/surface` | `inverseSurface` | Flat camelCase — NOT `inverse.surface` |
| `inverse/onSurface` | `inverseOnSurface` | Flat camelCase |
| `inverse/primary` | `inversePrimary` | Flat camelCase |

> Kai must use `token-name-mapping.json` for these — never infer the Panda token from the Figma path by simple slash→dot conversion. The mismatches above will produce wrong CSS custom property names.

### `dist/figma-codex.json`

**Generated by:** `pnpm build` (via `src/figma-codex/` scripts)

**Purpose:** Maps every DDS component's Figma node ID to its React import path, anatomy, props, variants, and usage example. This is the Code Connect substitute — Kai's component resolution layer.

**Structure (per component):**
```json
{
  "Button": {
    "name": "Button",
    "type": "simple",
    "figma": {
      "fileKey": "GaHmFfmvO4loUzuZS4TgEz",
      "nodeId": "930:XXXX",
      "url": "https://www.figma.com/design/GaHmFfmvO4loUzuZS4TgEz/..."
    },
    "imports": {
      "primary": "import { Button } from '@discourser/design-system/Button'",
      "namedExports": ["Button"],
      "subpath": "@discourser/design-system/Button"
    },
    "props": [
      { "name": "variant", "type": "string", "values": ["solid","surface","subtle","outline","plain","elevated"] },
      { "name": "size",    "type": "string", "values": ["2xs","xs","sm","md","lg","xl","2xl"] },
      { "name": "colorPalette", "type": "string", "values": ["primary","secondary","tertiary","error","neutral","gray","red"] }
    ],
    "subComponents": [],
    "example": "<Button variant=\"solid\" colorPalette=\"primary\">Label</Button>",
    "sourcePath": "src/components/Button.tsx"
  }
}
```

**For compound components**, `subComponents` lists all named parts:
```json
"subComponents": [
  { "name": "Root",    "element": "root" },
  { "name": "Trigger", "element": "trigger" },
  { "name": "Content", "element": "content" }
]
```

**Figma nodeId → component resolution:** When `get_design_context` identifies a component instance with a node ID, Kai looks it up in `figma-codex.json` by `figma.nodeId` to get the correct import path and props.

## Step 7: Panda CSS — Runtime Token System

**Command:** `pnpm prepare` (regenerates `styled-system/`)

The Panda CSS pipeline transforms all token definitions into:

1. **CSS custom properties** — defined in `:root` and `[data-theme=dark]`
2. **Utility classes** — generated for every token × property combination
3. **Recipe classes** — generated for every variant combination
4. **Type definitions** — TypeScript types for all token paths

**Key generated files:**
- `styled-system/tokens/index.d.ts` — TypeScript types for all valid token paths
- `styled-system/css/index.d.ts` — CSS utility function types
- `styled-system/recipes/` — Per-recipe type exports

**Token path resolution at runtime:**
```
textStyle="bodyLarge"
  → styled-system generates .textStyle_bodyLarge { font-size: var(--font-sizes-body-large); line-height: ... }

bg="surface.container"
  → styled-system generates .bg_surface\.container { background: var(--colors-surface-container) }
  → :root { --colors-surface-container: #EEEFE3 }
  → [data-theme=dark] { --colors-surface-container: #1E201A }
```

## Step 8: Kai's Spec Output

Kai (the Design Engineer Agent in Discourser.ai's BMAD workflow) uses all of the above to produce a **Design Implementation Spec** for Amelia (the Dev Agent).

### Kai's Three-Tier Resolution Strategy

1. **`figma-codex.json` (primary)** — If the Figma node matches a known component by `nodeId`, use the codex's import path, props, and anatomy. Highest fidelity.

2. **Storybook MCP (secondary)** — If the codex entry exists but has sparse `props` or empty `subComponents`, query Storybook MCP for the component's story to fill gaps.

3. **Knowledge fragments (fallback)** — For tokens and layout not covered by the above, use the Figma Translation docs (01–07) and skill files.

### What a Spec Looks Like

```markdown
## Component: ConversationHeader

**Source node:** 66:2739 (Figma)

### Resolved Component Tree

\`\`\`tsx
import { Breadcrumb } from '@discourser/design-system/Breadcrumb';
```

## Maintenance: Keeping the Pipeline in Sync

### After changing token values (rebrand, color update)

1. Update `docs/material-theme.json` from Material Theme Builder
2. Update `src/languages/material3.language.ts` (manually or via `pnpm transform:dtcg-to-contract`)
3. Run `pnpm figma:export` — regenerates all four export files including `token-name-mapping.json`
4. Import into Figma via `figma-token-sync` plugin
5. Run `pnpm prepare` — regenerates `styled-system/`
6. Bump version, publish package

### After adding a new component

1. Create component in `src/components/`
2. Create `src/figma-codex/{ComponentName}.figma.tsx` with node ID mapping
3. Run `pnpm build` — regenerates `dist/figma-codex.json`
4. Update `05-Components.mdx` in figma-translation
5. Bump version, publish package

### After adding a new semantic token

1. Add to `material3.language.ts` semantic + semanticDark
2. Add to `src/preset/semantic-tokens.ts`
3. Run `pnpm figma:export` — adds to `token-name-mapping.json`
4. Run `pnpm prepare`
5. Update `01-Colors.mdx` and `token-name-mapping.md`

### After changing typography weights

1. Update `material3.language.ts` typography scale
2. Run `pnpm figma:export` — regenerates `figma-text-styles.json` + updates `token-name-mapping.json` text style entries
3. Update in Figma via figma-token-sync
4. Update `02-Typography.mdx` and `src/stories/foundations/Typography.mdx`
