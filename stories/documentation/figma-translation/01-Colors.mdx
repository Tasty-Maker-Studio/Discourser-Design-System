# Color Translation Guide

This page translates color tokens from Figma, Tailwind CSS, and Shadcn/UI to Discourser Design System equivalents.

## Token Vocabulary: Two Valid Systems

Discourser exposes **two sets** of semantic color tokens. Both are real and valid — pick one system and stay consistent within a component.

| System | Source | Example |
|---|---|---|
| **M3 Semantic** | Material Design 3 color roles | `bg="surface.container"`, `color="onSurface"` |
| **Park UI Aliases** | Park UI compatibility layer | `bg="canvas"`, `color="fg.default"` |

The `globalCss` in `panda.config.ts` uses Park UI aliases (`bg: 'canvas'`, `color: 'fg.default'`) for the page root. Components and layout code should use **M3 Semantic tokens**.

## Color Palette System

Discourser uses the **colorPalette prop pattern** for theming components.

### Rule: Components First, Then Direct Tokens

- **If a component supports `colorPalette` prop** → ALWAYS use `colorPalette`
- **For custom styling outside components** → Use semantic tokens directly (`primary`, `error`, etc.)

### Available Palettes

- `primary` — Brand primary color
- `secondary` — Brand secondary color
- `tertiary` — Brand tertiary color
- `error` — Error/destructive actions
- `neutral` / `gray` — Neutral grayscale
- `red` — Red (for explicit red needs, distinct from `error`)

### Radix Color Scale Explained

Discourser uses the **Radix Colors scale** (1-12) for all color palettes, bridged to M3 semantic names:

| Scale | Usage | Example |
|---|---|---|
| `1` | App backgrounds | Lightest, near-white |
| `2` | Subtle backgrounds | Slightly tinted |
| `3` | UI element backgrounds | Hover states |
| `4` | Hovered UI backgrounds | Active subtle states |
| `5` | Active/selected states | Solid subtle |
| `6` | Borders | Default border color |
| `7` | Hovered borders | Interactive borders |
| `8` | Solid backgrounds | Strong solid color |
| `9` | **Solid colors (default)** | **Primary action color** |
| `10` | Hovered solid | Hover on buttons |
| `11` | Text colors | Low-contrast text |
| `12` | High-contrast text | Black/white text |

**Alpha variants** (`a1`–`a12`) have transparency, useful for overlays and subtle tints.

### When to Use colorPalette vs Direct Token

```tsx
// ✅ CORRECT: Component with colorPalette
<Button colorPalette="primary">Submit</Button>
// Generated: bg=primary, color=onPrimary, hover states auto-handled

// ✅ CORRECT: Custom Box needing primary accent background
<Box bg="primary.container" borderColor="primary">
  <Text color="onPrimary.container">Custom styled box</Text>
</Box>

// ❌ WRONG: Manual coloring when component supports colorPalette
<Button bg="primary" color="onPrimary">Submit</Button>
// Loses automatic hover states, contrast, and theme consistency
```

## Common Mistakes

### ❌ Hardcoded Hex Values

```tsx
// ❌ WRONG: Hardcoded hex
<Box style={{ backgroundColor: '#4C662B', color: '#FFFFFF' }}>
  Brand section
</Box>
```

**Why wrong:** Not theme-aware, doesn't support dark mode, not swappable if design language changes.

```tsx
// ✅ CORRECT: Semantic token
<Box bg="primary" color="onPrimary">
  Brand section
</Box>
```

---

### ❌ Using Tailwind Class Names

```tsx
// ❌ WRONG: Tailwind class
<Box className="bg-primary text-white">
  Content
</Box>
```

**Why wrong:** Discourser uses Panda CSS, not Tailwind. Classes won't work.

```tsx
// ✅ CORRECT: Panda CSS props
<Box bg="primary" color="onPrimary">
  Content
</Box>
```

---

### ❌ Manual Colors on colorPalette Components

```tsx
// ❌ WRONG: Manual background/color when component supports colorPalette
<Button bg="primary" color="onPrimary" _hover={{ bg: "primary.10" }}>
  Submit
</Button>
```

**Why wrong:** Duplicates work the component already does, loses compound variant logic.

```tsx
// ✅ CORRECT: Use colorPalette prop
<Button colorPalette="primary">
  Submit
</Button>
```

---

### ❌ Using Raw Radix Scale for Body Text

```tsx
// ❌ WRONG: Direct Radix scale for body text
<Text color="gray.12">
  Body text content
</Text>
```

**Why wrong:** Not semantic — use `fg.default` (Park UI alias) or `onSurface` (M3 semantic) for theme-aware text.

```tsx
// ✅ CORRECT: Semantic alias
<Text color="fg.default">
  Body text content
</Text>

// ✅ ALSO CORRECT: M3 semantic
<Text color="onSurface">
  Body text content
</Text>
```

---

### ❌ Using Old `m3Primary` / `m3Secondary` Prefix

```tsx
// ❌ WRONG: Old prefixed token names
<Box bg="m3Primary" color="onM3Primary">...</Box>
<Box bg="m3Secondary" color="onM3Secondary">...</Box>
```

**Why wrong:** The `m3` prefix was removed. Tokens are now clean: `primary`, `onPrimary`, `secondary`, `onSecondary`.

```tsx
// ✅ CORRECT: Clean token names
<Box bg="primary" color="onPrimary">...</Box>
<Box bg="secondary" color="onSecondary">...</Box>
```
