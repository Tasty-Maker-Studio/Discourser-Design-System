# Component Mappings

This document maps every Discourser component from its Figma/Shadcn equivalent, including props, variants, import paths, and compound component anatomy.

**Critical:** Every import path, prop name, variant value, and compound anatomy is verified against actual source files in `src/components/`.

---

## 1. Interactive Elements

### Button

**Maps from:** Material Design 3 Button, Shadcn Button
**Import:** `import { Button } from '@discourser/design-system'`
**Type:** Simple (recipe)

**Variants:**

| Source (Shadcn) | Discourser | Notes |
|---|---|---|
| `default` | `variant="solid"` | Filled background with solid color |
| `outline` | `variant="outline"` | Transparent background with border |
| `ghost` | `variant="plain"` | Transparent background, no border |
| `destructive` | `variant="solid" colorPalette="error"` | **Color is prop, not variant** |
| `secondary` | `variant="subtle"` | Subtle filled background |
| `link` | `variant="plain"` | Text-only appearance |
| - | `variant="surface"` | Surface background with border |
| - | `variant="elevated"` | Surface background with shadow elevation |

**Sizes:**

| Shadcn | Discourser | Height |
|---|---|---|
| `sm` | `size="sm"` | 36px (9) |
| `default` | `size="md"` | 40px (10) |
| `lg` | `size="lg"` | 44px (11) |
| - | `size="2xs"` | 24px (6) |
| - | `size="xs"` | 32px (8) |
| - | `size="xl"` | 48px (12) |
| - | `size="2xl"` | 64px (16) |

**Props:**

| Source Prop | Discourser Prop | Values | Notes |
|---|---|---|---|
| `variant` | `variant` | `solid \| surface \| subtle \| outline \| plain \| elevated` | Default: `solid` |
| `size` | `size` | `2xs \| xs \| sm \| md \| lg \| xl \| 2xl` | Default: `md` |
| `disabled` | `disabled` | `boolean` | Disables interaction |
| - | `colorPalette` | `primary \| secondary \| tertiary \| neutral \| error` | **Use for color theming** |
| - | `loading` | `boolean` | Shows loading spinner |
| - | `loadingText` | `ReactNode` | Text to show while loading |
| - | `spinnerPlacement` | `start \| end` | Spinner position (default: `start`) |

**See:** [Guidelines/99-Button](/docs/documentation-guidelines-99-button--docs)

**Maps from:** Shadcn Input with addons/icons

**Type:** Compound (slot recipe)

**Anatomy:**
- `<InputGroup.Root>` — Container for input with positioned elements


```tsx
<InputGroup startElement={<SearchIcon />}>

</InputGroup>
```

| Right icon | `endElement={<Icon />}` | Positioned at end (right in LTR) |

### Select

**Maps from:** Shadcn Select
**Import:** `import { Select } from '@discourser/design-system'`
**Type:** Compound (slot recipe)

**Anatomy:**
- `<Select.Root>` — Container managing select state
- `<Select.Label>` — Descriptive label
- `<Select.Control>` — Container for trigger button
- `<Select.Trigger>` — Button that opens/closes dropdown
- `<Select.ValueText>` — Displays selected value(s)
- `<Select.IndicatorGroup>` — Container for trigger indicators
- `<Select.Indicator>` — Chevron icon showing dropdown state
- `<Select.Positioner>` — Positions dropdown relative to trigger
- `<Select.Content>` — Dropdown container with animations
- `<Select.List>` — Scrollable list of options
- `<Select.Item>` — Individual selectable option
- `<Select.ItemText>` — Text content of option
- `<Select.ItemIndicator>` — Check icon shown on selected items
- `<Select.ItemGroup>` — Groups related options
- `<Select.ItemGroupLabel>` — Label for option group
- `<Select.ClearTrigger>` — Optional clear selection button
- `<Select.HiddenSelect>` — Hidden native select for form integration

**Shadcn → Discourser Mapping:**

| Shadcn | Discourser | Notes |
|---|---|---|
| `<Select>` | `<Select.Root>` | |
| `<SelectTrigger>` | `<Select.Control><Select.Trigger>` | Trigger must be wrapped in Control |
| `<SelectValue />` | `<Select.ValueText />` | |
| `<SelectContent>` | `<Select.Positioner><Select.Content>` | Needs Positioner wrapper |
| `<SelectItem>` | `<Select.Item>` | |
| `<SelectGroup>` | `<Select.ItemGroup>` | |
| `<SelectLabel>` | `<Select.ItemGroupLabel>` | For group labels, not field labels |

**Critical Differences:**
- Shadcn `<SelectContent>` → Discourser requires `<Select.Positioner><Select.Content>` nesting
- Shadcn `<SelectLabel>` (group label) ≠ `<Select.Label>` (field label)
- Discourser requires `<Select.HiddenSelect />` for form submission

**Variants:**

| Discourser | Visual |
|---|---|
| `variant="outline"` | Transparent background with border (default) |
| `variant="surface"` | Subtle background with border |

**Sizes:** `xs`, `sm`, `md` (default), `lg`, `xl`

**See:** [Guidelines/99-Select](/docs/documentation-guidelines-99-select--docs)

### Toast

**Maps from:** Shadcn Toast (Sonner)
**Import:** `import { toaster, Toaster } from '@discourser/design-system'`
**Type:** Programmatic API (not component-based)

**Shadcn → Discourser:**

| Shadcn | Discourser | Notes |
|---|---|---|
| `toast("Message")` | `toaster.create({ title: "Message" })` | Different invocation API |
| `toast.success("Done")` | `toaster.create({ type: "success", title: "Done" })` | Type is object property |
| `toast.error("Failed")` | `toaster.create({ type: "error", title: "Failed" })` | |
| `toast.loading("Wait")` | `toaster.create({ type: "loading", title: "Wait" })` | Shows spinner |

**Setup:**
```tsx
// Add to app root
import { Toaster } from '@discourser/design-system';

<Toaster />
```

**API:**
```tsx
toaster.create({
  title: "Title",
  description: "Description",
  type: "success" | "error" | "warning" | "loading",
  duration: 5000,
  closable: true,
  action: { label: "Undo", onClick: () => {} }
})
```

**Critical Differences:**
- Shadcn uses `toast()` function; Discourser uses `toaster.create()` method
- Shadcn uses `toast.success()`; Discourser uses `type` property
- Requires `<Toaster />` component in app root

**See:** [Guidelines/99-Toast](/docs/documentation-guidelines-99-toast--docs)

## 4. Overlays

### Dialog

**Maps from:** Shadcn Dialog
**Import:** `import { Dialog } from '@discourser/design-system'`
**Type:** Compound (slot recipe)

**Anatomy:**
- `<Dialog.Root>` — Container managing dialog state
- `<Dialog.Trigger>` — Button to open dialog
- `<Dialog.Backdrop>` — Semi-transparent overlay (scrim)
- `<Dialog.Positioner>` — Centers dialog on screen
- `<Dialog.Content>` — Dialog panel content
- `<Dialog.Title>` — Dialog heading
- `<Dialog.Description>` — Dialog description text
- `<Dialog.Header>` — Top section wrapper
- `<Dialog.Body>` — Main content wrapper
- `<Dialog.Footer>` — Bottom section wrapper (actions)
- `<Dialog.CloseTrigger>` — Close button

**Shadcn → Discourser:**

| Shadcn | Discourser | Notes |
|---|---|---|
| `<Dialog>` | `<Dialog.Root>` | |
| `<DialogTrigger>` | `<Dialog.Trigger>` | |
| `<DialogContent>` | `<Dialog.Positioner><Dialog.Content>` | **Requires Positioner wrapper** |
| `<DialogHeader>` | `<Dialog.Header>` | |
| `<DialogTitle>` | `<Dialog.Title>` | |
| `<DialogDescription>` | `<Dialog.Description>` | |
| `<DialogFooter>` | `<Dialog.Footer>` | |

**Critical:** Shadcn `<DialogContent>` → Discourser `<Dialog.Positioner><Dialog.Content>` (needs positioner wrapper).

**Shadcn `<DialogHeader>` and `<DialogFooter>`:**
- In Shadcn these are custom layout wrappers
- Discourser provides `<Dialog.Header>` and `<Dialog.Footer>` as styled slots
- Alternatively, use `<VStack gap="1.5">` for header or `<HStack gap="3" justify="flex-end">` for footer if you need custom layouts

### Breadcrumb

**Maps from:** Shadcn Breadcrumb, HTML `<nav aria-label="breadcrumb">` pattern
**Import:** `import { Breadcrumb } from '@discourser/design-system'`
**Type:** Compound (slot recipe)

**Anatomy:**
- `<Breadcrumb.Root>` — `<nav aria-label="breadcrumb">` container
- `<Breadcrumb.List>` — `<ol>` ordered list of items
- `<Breadcrumb.Item>` — Individual `<li>` crumb entry
- `<Breadcrumb.Link>` — `<a>` link for navigable crumbs
- `<Breadcrumb.Separator>` — Chevron separator between items (default `›`)
- `<Breadcrumb.Ellipsis>` — Collapsed items placeholder (`...`)

**Variants:**

| Prop | Values | Default |
|---|---|---|
| `variant` | `plain`, `underline` | `plain` |
| `size` | `xs`, `sm`, `md`, `lg` | `md` |

**Shadcn → Discourser:**

| Shadcn | Discourser |
|---|---|
| `<BreadcrumbList>` | `<Breadcrumb.List>` |
| `<BreadcrumbItem>` | `<Breadcrumb.Item>` |
| `<BreadcrumbLink>` | `<Breadcrumb.Link>` |
| `<BreadcrumbSeparator>` | `<Breadcrumb.Separator>` |
| `<BreadcrumbEllipsis>` | `<Breadcrumb.Ellipsis>` |

**Usage:**
```tsx
<Breadcrumb.Root>
  <Breadcrumb.List>
    <Breadcrumb.Item>
      <Breadcrumb.Link href="/">Home</Breadcrumb.Link>
    </Breadcrumb.Item>
    <Breadcrumb.Separator />
    <Breadcrumb.Item>
      <Breadcrumb.Link href="/settings">Settings</Breadcrumb.Link>
    </Breadcrumb.Item>
    <Breadcrumb.Separator />
    <Breadcrumb.Item>Profile</Breadcrumb.Item>
  </Breadcrumb.List>
</Breadcrumb.Root>
```

The last `<Breadcrumb.Item>` is automatically styled as the current page (`color: fg.default`). Use `<Breadcrumb.Ellipsis>` when the path is too long to display in full.

## Common Mistakes

```tsx
// ❌ WRONG: Using flat Shadcn pattern for compound Discourser component
<Dialog>
  <DialogContent>...</DialogContent>
</Dialog>

// ✅ CORRECT: Using Ark UI compound pattern
<Dialog.Root>
  <Dialog.Trigger>Open</Dialog.Trigger>
  <Dialog.Backdrop />
  <Dialog.Positioner>
    <Dialog.Content>...</Dialog.Content>
  </Dialog.Positioner>
</Dialog.Root>
```

```tsx
// ❌ WRONG: Missing Positioner wrapper for Dialog/Drawer/Popover Content
<Dialog.Root>
  <Dialog.Content>...</Dialog.Content>
</Dialog.Root>

// ✅ CORRECT: Positioner wraps Content
<Dialog.Root>
  <Dialog.Positioner>
    <Dialog.Content>...</Dialog.Content>
  </Dialog.Positioner>
</Dialog.Root>
```

```tsx
// ❌ WRONG: Using "destructive" as a variant
<Button variant="destructive">Delete</Button>

// ✅ CORRECT: Using colorPalette for destructive intent
<Button variant="solid" colorPalette="error">Delete</Button>
```

```tsx
// ❌ WRONG: Using Shadcn SelectLabel for group labels
<Select.Root>
  <Select.Label>Filter by status</Select.Label>  {/* This is for the field */}
  <Select.ItemGroup>
    <Select.Label>Active</Select.Label>  {/* WRONG */}
  </Select.ItemGroup>
</Select.Root>

// ✅ CORRECT: Using ItemGroupLabel for option groups
<Select.Root>
  <Select.Label>Filter by status</Select.Label>  {/* Field label */}
  <Select.ItemGroup>
    <Select.ItemGroupLabel>Active</Select.ItemGroupLabel>  {/* Group label */}
  </Select.ItemGroup>
</Select.Root>
```

```tsx
// ❌ WRONG: Importing from wrong path
import { Button } from '@ark-ui/react'
import { button } from '../preset/recipes/button'

// ✅ CORRECT: Importing from design system
import { Button } from '@discourser/design-system'
```

```tsx
// ❌ WRONG: Using Shadcn CardContent
<Card.Root>
  <Card.Content>...</Card.Content>
</Card.Root>

// ✅ CORRECT: Using Card.Body (different name)
<Card.Root>
  <Card.Body>...</Card.Body>
</Card.Root>
```

```tsx
// ❌ WRONG: Missing Select.HiddenSelect (breaks form submission)
<Select.Root>
  <Select.Control>
    <Select.Trigger>
      <Select.ValueText />
    </Select.Trigger>
  </Select.Control>
  <Select.Positioner>
    <Select.Content>
      <Select.List>{/* items */}</Select.List>
    </Select.Content>
  </Select.Positioner>
  {/* Missing HiddenSelect */}
</Select.Root>

// ✅ CORRECT: Include HiddenSelect for forms
<Select.Root>
  {/* ... */}
  <Select.HiddenSelect />
</Select.Root>
```

```tsx
// ❌ WRONG: Using toast() function like Shadcn
import { toast } from '@discourser/design-system';
toast.success("Done");

// ✅ CORRECT: Using toaster.create() method
import { toaster } from '@discourser/design-system';
toaster.create({ type: "success", title: "Done" });
```

## Notes for Kai Design Engineer Agent

When translating from Figma/Shadcn to Discourser:

1. **Always check if component is compound** — Use namespace pattern for compound components
2. **Positioner wrapper is critical** — Dialog, Drawer, Popover, Tooltip, Select all require Positioner wrapping Content
3. **colorPalette is a prop, not a variant** — Don't confuse `variant="destructive"` with `colorPalette="error"`
4. **Shadcn component names may differ** — `CardContent` → `Card.Body`, `Sheet` → `Drawer`
5. **Import from package only** — `@discourser/design-system`, never from internal paths
6. **HiddenSelect/HiddenInput required** — For form components that need native input integration
7. **Toast uses programmatic API** — `toaster.create()` method, not `toast()` function
8. **Default variants matter** — Button defaults to `solid`, Card defaults to `outline`, Badge defaults to `subtle`
