# Design Tokens

## Directory Structure

### tokens/export-to-figma/

Generated by `pnpm figma:export`. These files are handed to the
figma-token-sync plugin to import into Figma. Not committed to git.

- `figma-variables.json` — Primitives + Semantic + Spacing & Shape variables
- `figma-text-styles.json` — Typography text styles (56 weight variants)
- `figma-effect-styles.json` — Elevation shadow levels

### tokens/imported-from-figma/

Generated by `pnpm organize-tokens` after running Export Variables in the
figma-token-sync plugin. Used to sync Figma changes back into DDS. Not
committed to git.

- `primitives-generated.json`
- `semantic-light-generated.json`
- `semantic-dark-generated.json`
- `figma-variables.json` — Combined token file (for re-importing to Figma)

### tokens/primitives/ and tokens/semantic/

Reference files committed to git. Source of truth for tonal palettes and
semantic color mappings.

## Workflows

### DDS → Figma (push tokens to Figma)

1. `pnpm figma:export`
2. Open figma-token-sync plugin in Figma
3. Import Variables: choose `tokens/export-to-figma/figma-variables.json`
4. Import Text Styles: choose `tokens/export-to-figma/figma-text-styles.json`

### Figma → DDS (pull changes from Figma)

1. Open figma-token-sync plugin in Figma
2. Click Export Variables → download files
3. `pnpm organize-tokens` (places files in `tokens/imported-from-figma/`)
4. Review changes, run `pnpm build:panda`
